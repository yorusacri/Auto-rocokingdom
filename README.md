# Auto-Roco 洛克王国自动化助手

> ⚠️ **游戏更新提示**：游戏最近的更新加强了对模拟输入的检测，涉及模拟输入的功能暂时无法使用。具体表现为：在游戏刚启动的几分钟内脚本能正常模拟输入，但很快就会失效（应该是游戏后台启动了输入过滤程序）。

这是一个基于图像识别（OpenCV）的游戏自动化辅助工具，专门为《洛克王国：世界》设计。它能够自动识别战斗状态，并根据用户选择执行"聚能模式""逃跑模式""污染计数模式"或"智能模式"。

## 🚀 核心功能
*   **聚能模式 (Mode 1)**：在进入战斗时，自动模拟键盘点击 `X` 键。
*   **逃跑模式 (Mode 2)**：在进入战斗时，自动按 `ESC` 键并点击"是"按钮。
    *   统一使用**物理鼠标点击**。
    *   执行逃跑点击时，请确保"是"按钮露出，不能被其他窗口遮挡。
*   **污染计数模式 (Mode 3)**：仅进行检测与计数，不执行任何自动操作。
    *   终端展示污染次数（内部仍保留战斗计数用于战斗区间识别）。
*   **智能模式 (Mode 4)**：每场战斗开始时进行一次类型判定，支持自定义配置。
    *   默认配置：污染战斗执行聚能（按 `X`），普通战斗执行逃跑（`ESC` + 点击"是"按钮）。
    *   可选行为：只聚能 / 逃跑 / 释放技能1后聚能（按 `1` 再按 `X`）。
    *   **技能1+聚能注意事项**：请将需要的技能放在一号技能位，并使用罗隐等不会因能量不足而放不出技能的精灵。
    *   运行时可按提示自定义两种战斗类型的行为。
*   **同行自动确认**：非战斗状态下，检测到重新同行请求（`qiudaidai.png`）时自动按 `F` 确认。
*   **污染精灵日志**：自动识别污染战斗中的精灵名称，记录到 `logs/pollute_log.csv`（含序号、时间、精灵名称），可用 Excel 直接打开查看。
    *   精灵名称识别为**可选功能**，需要额外安装 EasyOCR（见下方步骤）。未安装时精灵名称统一记录为"未知"，不影响其他功能使用。
*   **后台监控**：支持游戏窗口在被遮挡时依然进行图像识别（只要窗口不最小化）。
    *   **输入方式可选**：默认使用 `sendinput`（拟真度高，需游戏在前台）；切换为 `postmessage` 可后台发送指令（拟真度低）。通过 `config.py` 中的 `input_method` 配置。
    *   最近游戏官方加强了对脚本的检测。建议如果没有挂后台的需求，尽可能使用`sendinput`模式。

## 📖 使用步骤

### 第一步：安装 uv

uv 是本项目的包管理器，它会自动下载和管理 Python，**无需事先安装 Python**。

打开任意终端（按 `Win+R`，输入 `powershell`，回车），运行：

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

安装完成后，**关闭终端并重新打开**，输入 `uv --version`，出现版本号即表示安装成功。

### 第二步：以管理员身份打开终端并进入项目目录

脚本需要向游戏窗口发送键盘和鼠标指令，Windows 安全策略要求**必须以管理员身份运行**才能模拟输入。

1.  按 `Win+X`，选择 **Windows PowerShell (管理员)** 或 **终端 (管理员)**。
2.  在终端中 `cd` 到项目目录，例如：
    ```powershell
    cd C:\Users\你的用户名\Desktop\Auto-rocokindom
    ```
    > 小技巧：在文件资源管理器中打开项目文件夹，点击地址栏复制路径，直接粘贴到终端即可。

### 第三步：安装依赖

```bash
uv sync
```

### 第四步（可选）：安装精灵名称识别

如需自动识别污染精灵名称并记录到 CSV 日志，额外安装 EasyOCR：

```bash
uv sync --extra easyocr
```

> 为什么用 CPU 版 torch？精灵名称识别是轻量 OCR 任务，CPU 运算完全够用。CUDA 版 PyTorch 体积超过 2GB，且需要 NVIDIA 显卡。

跳过此步骤不影响其他功能，日志中的精灵名称将统一记录为"未知"。

> 首次运行时 EasyOCR 会自动从外网下载识别模型（约 100MB），请耐心等待。若下载缓慢，建议开启代理。

### 第五步：启动脚本

在管理员终端中运行：

```bash
uv run main.py
```

随后根据屏幕提示输入 `1`-`4` 选择模式。选择智能模式（4）时可按提示自定义污染战斗和普通战斗的行为。

## 💡 注意事项
*   **分辨率建议**：本脚本已实现**自适应分辨率**功能，能够根据游戏窗口大小自动缩放识别模板。
    *   **推荐分辨率**：**2K（2560x1600 或 2560x1440）**。分辨率越高，画面细节越丰富，识别精度越高。
    *   **精度提示**：在低分辨率下，由于像素损失，识别分数 (Score) 会降低。我们已将触发阈值调低至 **0.4** 以兼容低端设备。
    *   **进阶适配**：如果你的分辨率非常特殊导致脚本失效，你可以参考 `templates/` 目录下的截图格式，**在你自己的分辨率下重新截图**并覆盖 `templates/` 中的同名文件（如 `skill_main.png`），脚本将完美适配你的专属环境。
*   **停止方式**：在任何时候按下 **`Ctrl+C`** 即可安全退出。
*   **窗口状态**：游戏窗口**不能最小化**到任务栏，否则图像识别将失效。
    *   使用默认输入方式 `sendinput` 时，游戏窗口**必须在前台**，否则按键会发送到其他窗口。
    *   使用 `postmessage` 输入方式时，游戏窗口可被覆盖，按键直接投递到游戏窗口句柄，不要求前台。
*   **逃跑模式点击要求**：逃跑模式使用鼠标点击，请确保点击瞬间"是"按钮可见且未被遮挡。`sendinput` 方式下若被覆盖，点击会落在前景窗口上。
*   **污染日志**：每次污染战斗开始时，自动通过 OCR 识别右上角精灵名称并记录到 `logs/pollute_log.csv`。未安装 EasyOCR 时记录为"未知"。
