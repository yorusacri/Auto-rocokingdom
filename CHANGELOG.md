# 更新日志

### 2026-05-30

1. **Edge 浏览器兼容修复**：修复 `_TeeStdout.fileno()` 在 `console=False` 打包时抛出 `OSError` 导致无 Chrome 用户无法启动的问题。Edge 浏览器启动时会将 `sys.stdout` 传入 subprocess，subprocess 调用 `fileno()` 获取文件描述符，但 PyInstaller 无终端模式没有底层文件描述符。改为返回 `NUL` 的有效文件描述符作为兜底。
2. **测试基础设施**：添加 pytest 测试套件（81 个用例），覆盖 `_TeeStdout`、配置加载/保存、模板名规范、轮询间隔边界、日志分类、模式注册表、SmartMode 状态机、引擎快照和污染日志等模块。

### 2026-05-27

1. **全模块诊断日志**：所有模块的静默失败点已添加 `[诊断]` / `[警告]` / `[错误]` 前缀日志，启动或运行时报错会自动输出详细诊断信息到引擎日志面板，方便用户截图反馈。
   - 模板加载失败时自动输出系统环境（Python/OpenCV 版本、DLL 状态、文件完整性）
   - `cv2.imread` 失败时自动 fallback 到 Python I/O + `cv2.imdecode`，并报告原因
   - 截图模块（PrintWindow/BitBlt/位图不匹配）、输入模块（Interception 初始化/点击异常）、窗口模块（DPI 感知）等全链路静默错误现在都会输出诊断日志
   - 引擎线程异常现在包含完整 traceback，不再只输出异常消息
   - CLI 模式循环增加异常保护，捕获后自动恢复
   - 配置文件读写异常、CSV 日志写入异常、OCR 初始化异常均输出诊断信息
   - 正常启动时不再输出任何诊断信息，仅在出错时输出
2. stdout 全局接管：release 版（`console=False`）所有 `print()` 输出现在也会显示在 GUI 引擎日志面板中。
3. 引擎启动时验证配置引用的模板文件是否都已成功加载，缺失时输出警告。

### 2026-05-25

1. **桌面客户端 GUI**：使用 Eel 框架，暗色仪器面板风格界面，功能包括：
   - 一键暂停/恢复（头部大按钮 + 空格键快捷操作），暂停后显示红色停止按钮
   - 实时状态面板：战斗场次、污染次数、行动/结束/同行检测分数（保留两位小数）
   - 终端风格滚动日志：只显示程序行为（待机/触发/战斗中），不再输出模板名和原始分数
   - 交互式窗口选择器：刷新扫描所有匹配窗口，单选后热切换目标
   - 设置面板（齿轮图标）：匹配阈值滑块、检测间隔滑块，保存到 `user_prefs.json` 下次启动生效
   - 模式 + 行为选择自动持久化，下次打开 GUI 自动恢复
   - 运行中热切换行为选项和目标窗口，模式选择需停止后切换
   - 启动方式：`uv run main.py --gui`，打包为 `auto-roco-client.exe`（无终端窗口）
2. Engine 重构：提取 `tick()` 单次迭代接口，新增暂停/恢复/停止控制（`threading.Event`）、`switch_window()` 热切换窗口，GUI 与 CLI 模式共用核心逻辑。
3. 依赖新增 `eel`：轻量级 Python 桌面应用框架，前端使用 HTML/CSS/JS。
4. 代码清理：移除 `build.py`（改为 `.spec` 文件直接构建）、`tools/` 调试脚本目录、`template_debug/` 调试截图、`user_prefs.json`（加入 `.gitignore`）。`config.py` 移除 4 个无效配置项。
5. 移除 CLI 版本：产品线统一为桌面客户端，只保留 `auto-roco-client-lite`（精简）和 `auto-roco-client-full`（含 OCR 精灵识别）。`uv run main.py --gui` 用于开发调试。GitHub Actions 同步简化。
